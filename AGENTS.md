# Repository Agent Notes

## Purpose
This repository powers the ebook-tools platform, bundling a FastAPI backend, background job helpers, and a Vite/React frontend for rendering and managing multimedia ebook assets.

## Key Entry Points
- **Backend:** `modules/webapi/application.py` exposes the FastAPI app factory referenced by `python -m modules.webapi` and the `ebook-tools-api` console script.
- **CLI / Orchestration:** `ebook-tools.py` and `scripts/` house helper commands for running pipelines locally.
- **Frontend:** `web/` contains the Vite application. Source files live under `web/src/`, while built assets default to `web/dist/`.
- **Offline exports:** `modules/services/export_service.py` builds offline player bundles for completed jobs/library entries via `/api/exports`, using the export player entry at `web/export.html` + `web/src/export-player/` (built with `vite build --mode export` into `web/export-dist`, override via `EBOOK_EXPORT_PLAYER_ROOT`). Export archives land under `storage/exports` (override via `EBOOK_EXPORT_ROOT`).
- **Apple Clients:** `ios/InteractiveReader/` now ships iOS/iPadOS + tvOS targets that authenticate via `/api/auth/login`, list Library entries from `/api/library/items`, and play library media via `/api/library/media/{job_id}` plus `/api/jobs/{job_id}/timing`. Shared SwiftUI flows live under `Features/Auth`, `Features/Library`, and `Features/Playback`, with the tvOS target configured through `InteractiveReader/Supporting/Info-tvOS.plist`. Apple Create source pickers use `/api/pipelines/files` for newest-first EPUBs, `/api/acquisition/discover` for shared local EPUB and NAS video discovery, `/api/subtitles/sources` for recursive preferred subtitle sources (SRT/VTT first, newest modified first, ASS after), and `/api/subtitles/youtube/library` for newest-first NAS video dubbing sources; the EPUB/subtitle source picker routes emit token-safe aggregate duration telemetry via `ebook_tools_source_picker_route_duration_seconds`. tvOS exposes the native Create tab and all Create modes via server/NAS-backed sources while local file import remains iOS/iPadOS-only. Narrate EPUB can delete selected backend EPUB sources via `DELETE /api/pipelines/files`, follows visible symlinked NAS collection folders through the shared source-discovery helper, and accepts zero-based backend chapter indexes as sentence-1 chapter selections when loading `/api/pipelines/files/content-index`; subtitle jobs remember the Show Original preference per API/user scope and can delete selected backend subtitle sources via `/api/subtitles/delete-source`; YouTube dubbing defaults to the newest playable NAS video/subtitle pair, supports a remembered NAS base directory, and restores the last valid NAS video/subtitle selection per API/user scope, Apple Create remembers shared language and lookup-cache defaults per API/user scope, keeps EPUB and YouTube Dub acquisition discovery state isolated per media kind, and Create history defaults reuse prior settings, generated-book source context, and voice overrides without replacing backend-selected NAS EPUB/subtitle/video paths. Apple Create also lists and deletes saved templates from `/api/creation/templates`; generated-book/Narrate EPUB templates apply source, language, narration, output, image, metadata, and worker controls, while subtitle/YouTube templates apply source, language, model, timing, output, metadata JSON, and tuning controls, all preserving the edited-field guard against later defaults. Its narration pickers load `/api/audio/voices` and preview voices through `/api/audio`.
- **Apple Resume Sync:** Cross-device resume positions for library items are stored in the CloudKit private database for container `iCloud.com.ebook-tools.interactivereader` (keyed by username + job id), with a local UserDefaults fallback for offline playback.
- **Apple resume state:** Apple clients persist last-played positions in iCloud key-value storage (per user + job) so opening a library item offers resume vs restart across iPhone/iPad/Apple TV.
- **Apple offline sync:** iOS/iPadOS can download completed jobs or library items into iCloud Drive (`Documents/OfflineMedia/<kind>/<job_id>`) for offline playback, with per-row sync/remove controls in job and library lists.
- **Playback bookmarks:** Web and Apple players support per-job bookmarks with add/jump/remove controls in Interactive/Video playback; when authenticated they sync via `/api/bookmarks/{job_id}` (stored under `storage/bookmarks/<user>/<job>.json`), with web `localStorage` and Apple `UserDefaults` as local/offline caches. Playback resume positions sync through `/api/resume/{job_id}`, and Web Library rows can batch-fetch current user resume evidence with `GET /api/resume?job_id=<id>` while still falling back to the existing per-job `media-memory:<job_id>` session cache.
- **Frontend Book Flows:** `web/src/pages/NewImmersiveBookPage.tsx` unifies the submission and settings experience behind the sidebar entry “Narrate Ebook,” sharing language state with `web/src/pages/CreateBookPage.tsx` and `web/src/components/book-narration/BookNarrationForm.tsx` via `web/src/context/LanguageProvider.tsx`. `CreateBookPage` now submits a managed `book` job via `/api/books/jobs`, first generating a seed EPUB with the LLM and then running the standard pipeline with the same Narrate Ebook form settings. `/api/books/options` supported input/output language lists are passed into the reusable narration form so Web Create pickers stay aligned with backend-supported language catalogs instead of relying only on bundled defaults. `/api/pipelines/files` returns optional `size_bytes` / `modified_at` metadata, recursively discovers EPUBs in visible books-root subfolders, and sorts EPUBs newest-first so Web and Apple untouched source defaults load the latest backend-visible NAS ebook after any preferred sample rule. Web Narrate Ebook also offers a Discovery sources dialog (`BookNarrationDiscoveryDialog` + `useBookNarrationDiscovery`) backed by `/api/acquisition/discover`; selecting a local EPUB candidate only fills the existing `input_file` field so submission payloads and templates remain unchanged. Web Video Dubbing and Apple YouTube Dub use the same discovery contract for backend-visible NAS video candidates, filling existing video/subtitle selections without changing dub payloads.
- **Source discovery:** `modules/services/source_discovery.py` centralizes NAS-safe visible-file walking for backend picker routes. Use it when adding source pickers so hidden folders are pruned, visible symlinked collection folders are followed with cycle protection, suffix filters are case-insensitive, vanished candidates are skipped, transient root scans return empty results, and cached stat metadata is reused instead of open-coding `os.walk`/`Path.stat` loops.
- **Acquisition providers:** `modules/services/acquisition/` owns the token-safe discovery/acquisition provider registry exposed by `/api/acquisition/providers` (`modules/webapi/routers/acquisition.py`). It lists configured/planned lawful source providers for local EPUBs, NAS videos, YouTube URL/search, Download Station/indexer handoff, and public/open ebook catalogs without returning credentials, raw provider tokens, or scraping warmed browser state. `GET /api/acquisition/discover` returns normalized editor/admin-only candidates for backend-visible EPUBs, Project Gutenberg/Gutendex public catalog matches, NAS videos with subtitle hints, and configured YouTube Data API metadata search; actual downloader/acquire handoff remains a separate reviewed step. The public runtime descriptor advertises providers as `creation.acquisitionProvidersPath` and discovery as `creation.acquisitionDiscoverPath`.
- **Search:** `modules/webapi/routes.py` exposes `/api/pipelines/search`, which now requires a `job_id` query parameter, scans that job's persisted chunks for snippets surfaced in the frontend `MediaSearchPanel`, and falls back to library metadata when the pipeline job has already been archived.
- **Audio Voices:** `modules/webapi/routers/audio.py` serves `/api/audio/voices` for language-specific voice inventories and `/api/audio` for preview synthesis consumed by the `BookNarrationForm` voice picker.
- **LLM Providers:** `/api/pipelines/llm-models` returns provider-prefixed model identifiers (`ollama_local`, `ollama_cloud`, `lmstudio_local`), and LM Studio uses the OpenAI-compatible endpoint configured via `lmstudio_url` / `LMSTUDIO_URL`.
- **Library:** `modules/library/` now splits the domain into `library_models`, `library_repository`, `library_metadata`, `library_sync`, and the lightweight `library_service` facade consumed by `modules/webapi/routers/library.py`; the frontend experience lives in `web/src/pages/LibraryPage.tsx` and related components.
- **Library video items:** Library entries track an `item_type` (`book` or `video`); completed YouTube dubbing jobs (`youtube_dub`) can be moved into the Library NAS for playback alongside books.
- **Metadata Storage:** `storage/<job_id>/metadata/` now persists a compact `job.json` manifest plus per-chunk JSON files (`chunk_0000.json`, `chunk_0001.json`, …) for highlight timelines. Chunk data is available via `generated_files.chunks[]` in `job.json`. Use `MetadataLoader` in `modules/metadata_manager.py` to read either the new chunked format or legacy single-file payloads.
- **Content Index:** `storage/<job_id>/metadata/content_index.json` stores chapter ranges (title + sentence boundaries + alignment status). `book_metadata` includes `content_index_path` / `content_index_url` pointers so downstream tools can fetch it for chapter navigation or LLM analysis.
- **Sentence Images:** `modules/images/` provides the Draw Things / Stable Diffusion client (`drawthings.py`) and prompt helpers (`prompting.py`). `modules/core/rendering/pipeline.py` runs image generation in parallel with translation when `add_images` is enabled, storing PNGs under `media/images/<range_fragment>/sentence_XXXXX.png` and writing paths/prompts into chunk metadata. The web API exposes inspection/regeneration endpoints in `modules/webapi/routes/media_routes.py`, and the frontend consumes them via the image reel in `web/src/components/InteractiveTextViewer.tsx` plus the `MyPainter` UI (`web/src/components/MyPainterAssistant.tsx`).
- **Subtitles:** Subtitle parsing/rendering now lives in `modules/subtitles/` (I/O + language helpers + merging + rendering + translation utilities) with `modules/subtitles/processing.py` orchestrating colourised output. Supports per-word SRT highlighting, optional ASS exports, configurable start/end time windows (including relative offsets), and the `original_language`/`show_original` options exposed on `SubtitleToolPage.tsx` via `modules/webapi/routers/subtitles.py`. Web Subtitle Tool can save sanitized `subtitle_job` creation templates through `/api/creation/templates` for Apple Create reuse.
- **YouTube NAS dubbing:** `modules/services/youtube_dubbing/` splits NAS discovery, subtitle extraction, dubbing generation, and service orchestration; it lists downloaded videos under `/Volumes/Data/Download/DStation`, pruning hidden folders/files during discovery, and builds dubbed audio tracks from adjacent ASS subtitles. `/api/subtitles/youtube/dub` enqueues `youtube_dub` jobs (tracked alongside pipelines/subtitles) via `YoutubeDubbingService`, and the frontend entry `web/src/pages/VideoDubbingPage.tsx` lists NAS videos plus active dubbing jobs. Web Video Dubbing can save sanitized `youtube_dub` creation templates through `/api/creation/templates` for Apple Create reuse.
- **Alignment:** `modules/align/backends/whisperx_adapter.py` shells out to the WhisperX CLI when `forced_alignment_enabled` and `alignment_backend` are configured, allowing `modules/render/audio_pipeline.py` to pull real word tokens whenever the TTS backend omits them. The pipeline now auto-selects the WhisperX model per target language (honouring `alignment_model_overrides` when provided) and logs the choice so drift investigations stay transparent. Set `EBOOK_HIGHLIGHT_POLICY=forced` to fail jobs that would otherwise fall back to inferred timings.
- **Timing Policies:** Use `char_weighted_highlighting_default=true` (or `EBOOK_CHAR_WEIGHTED_HIGHLIGHTING_DEFAULT=1`) to prefer character-weighted timings and pair it with `char_weighted_punctuation_boost=true` (`EBOOK_CHAR_WEIGHTED_PUNCTUATION_BOOST=1`) when you want punctuation-aware pacing before the renderer falls back to uniform inference.

## Audio, Metadata & Highlighting Cheat Sheet
- **Audio generation.** `modules/render/audio_pipeline.py` streams translation tasks into the active backend from `modules/audio/backends/`. Each sentence now yields separate `orig` and `translation` audio tracks (no concatenated mix), and the worker records provenance for both via `highlighting_summary` (translation) plus `original_highlighting_summary` (original), covering backend tokens, WhisperX (`modules/align/backends/whisperx_adapter.py`), char-weighted inference, or uniform fallbacks.
- **Metadata creation.** `modules/services/job_manager/persistence.py` writes `metadata/job.json` and per-chunk files (`metadata/chunk_XXXX.json`). Chunk data is accessed via `generated_files.chunks[]` in `job.json`. Chunks now carry `audioTracks.orig` + `audioTracks.translation` and `timingTracks.original` + `timingTracks.translation` (legacy `orig_trans`/`mix` only for older jobs). `book_metadata` is populated with `input_language`/`original_language` plus `target_language`/`translation_language`/`target_languages`, and it now records `translation_provider`/`translation_model` alongside `transliteration_mode`/`transliteration_model`/`transliteration_module` so clients can label tracks, pick voices, and trace translation strategy. `MetadataLoader` in `modules/metadata_manager.py` can load both the chunked format and the legacy single-file payload, so always run metadata reads through it to stay forward-compatible.
- **Fallback tracking.** When translation or TTS fall back mid-job, `generated_files` now captures `translation_fallback` / `tts_fallback` (trigger + reason + fallback model/voice) so metadata and job cards can surface the switch. Configure defaults with `EBOOK_TRANSLATION_FALLBACK_MODEL`, `EBOOK_TRANSLATION_LLM_TIMEOUT_SECONDS`, and `EBOOK_TTS_FALLBACK_VOICE`.
- **Highlighting controls.** `EBOOK_HIGHLIGHT_POLICY` decides whether a job may fall back to inferred timings, while `char_weighted_highlighting_default`/`char_weighted_punctuation_boost` (or their env vars) tune character-weighted inference. Each chunk now records a `highlighting_policy` summary (per-token `policy`/`source` are no longer persisted); legacy jobs still surface policy via `/api/jobs/{job_id}/timing`.
- **Sentence images.** When `add_images` is enabled, the renderer calls a Draw Things / Stable Diffusion `txt2img` endpoint (`image_api_base_url` or clustered `image_api_base_urls`) in parallel worker threads (`image_concurrency`, default 4), balancing work by node availability so faster nodes naturally handle more. Each generated image is stored under `storage/jobs/<job_id>/media/images/<range_fragment>/sentence_XXXXX.png` and referenced from `metadata/chunk_XXXX.json` as `sentences[].image` (plus `image_path` / `imagePath`). The default `prompt_plan` pipeline builds a batched LLM prompt plan (`metadata/image_prompt_plan.json`). The `visual_canon` pipeline persists `metadata/visual_canon.json` plus per-scene files under `metadata/scenes/` and renders per-sentence `img2img` for continuity. Use `docs/sentence_images.md` as the reference for prompt/style rules and regeneration endpoints.
- **ASS word highlights.** Subtitle exports pace highlights per word (character-weighted with a light uniform blend) across the subtitle span, cap preroll at 0.35s, and drop tail padding to avoid overlaps; YouTube dubbing ignores noisy speech windows and trusts the dubbed subtitle duration so karaoke timing stays smooth even when silence detection jitters.
- **Update expectation.** Whenever the audio stack, metadata layout, or highlighting logic changes, update these bullets plus the relevant docs (`README.md`, `docs/sentence_highlighting.md`, `docs/interactive_reader_metadata.md`) so downstream agents do not need to reverse-engineer the current behaviour.
  When sentence-image generation or playback UX changes, update `docs/sentence_images.md` and `docs/frontend-sync.md` as well.

## Common Workflows
- Create a virtual environment and install dependencies with `pip install -e .[dev]`.
- Apple builds run on a separate MacBook Air while Codex runs on a Mac Studio; expect different absolute paths between machines.
- Start the API for local development with `uvicorn modules.webapi.application:create_app --factory --reload`.
- `scripts/run-webapi.sh` starts a second HTTP listener for Apple TV on port 8001 by default; set `EBOOK_API_TV_HTTP_PORT=0` (optional `EBOOK_API_TV_HTTP_HOST`) to disable or override.
- Local Mac iPad-style Apple builds use `make build-apple-macos-ipad-style`; `make apple-macos-ipad-destination` and `make build-apple-macos-ipad-style-dry-run` inspect the resolved destination/build command without physical device deployment.
- Office-iPad-only local Apple gates use `make build-apple-office-ipad-surfaces` and `make verify-apple-office-ipad-surfaces` to compile the iPad simulator plus local Mac iPad-style app without depending on an iPhone or physical-device deployment.
- Apple Settings and Create readiness surface `/api/system/runtime` Create, Library action, offline export, and playback-state contracts; keep `settingsCreateContractRow`, `settingsLibraryActionsContractRow`, `settingsOfflineExportsContractRow`, and `settingsPlaybackStateContractRow` aligned with the Swift API client constants before deploying to hardware.
- Shared physical-device pipeline wrappers are `make apple-device-preflight APPLE_DEVICE_PROFILE=ipad APPLE_DEVICE_ID=<id>`, `make apple-device-signed-build-only APPLE_DEVICE_PROFILE=ipad`, and `make apple-device-deploy-dry-run APPLE_DEVICE_PROFILE=appletv`; only explicit user deploy requests should remove dry-run/confirmation guards.
- The reusable Apple device deploy helper invokes `src/check_poc_readiness.py` when readiness is enabled. In ebook-tools this legacy-named hook is token-safe and verifies `/_health` plus `/api/system/runtime` Create, Library action, offline export, and playback-state metadata for unattended iPad/TV deploy gates.
- Unattended physical-device deploy dry-runs use `scripts/apple_unattended_device_update.sh --profile iphone|ipad|appletv`; the helper resolves friendly device names to hardware UDIDs before calling `xcodebuild`, and the `appletv` profile selects `InteractiveReaderTV`, `com.example.InteractiveReader.tvos`, and `Debug-appletvos/InteractiveReaderTV.app` without manual environment overrides. Remove dry-run/install guards only after an explicit user deploy request.
- If Xcode CLI automatic signing cannot use the signed-in account for a physical iPhone/iPad deploy but local Xcode-managed profiles already contain the full iCloud/Sign in with Apple/Push capability set, preserve entitlements with the repo-owned planner: `make apple-device-full-entitlement-plan APPLE_DEVICE_ID=<id> FULL_CAPABILITY_IOS_PROFILE=<app.mobileprovision> WILDCARD_IOS_EXTENSION_PROFILE=<extension.mobileprovision> APPLE_DEVELOPMENT_IDENTITY="<identity>"`. It prints the unsigned `generic/platform=iOS` build, app/extension profile embedding, `scripts/apple_merge_entitlements.py` generation steps, nested dylib/extension signing, app signing, verification, and guarded `apple_unattended_device_update.sh --skip-build --install --launch --launch-console-timeout 10` handoff. Use `make apple-device-full-entitlement-build ...` to run build/sign/verify without installing. Use `CONFIRM_PHYSICAL_DEVICE_UPDATE=YES make apple-device-full-entitlement-install ...` only after an explicit hardware deploy request. For successful install verification, sign the app/extension with merged entitlements that include the profile-generated `application-identifier`, team id, `get-task-allow`, and keychain groups plus the app iCloud/Push/Sign in with Apple entitlements; signing only with `InteractiveReader.entitlements` can pass local `codesign --verify` but fail device install. If a current signed full-entitlement app already exists and the user asks for the latest stable hardware deploy, prefer `CONFIRM_PHYSICAL_DEVICE_UPDATE=YES make apple-device-full-entitlement-stable-install APPLE_DEVICE_ID=<id> APPLE_DEVICE_SIGNED_ARTIFACT_PATH=<InteractiveReader.app>`; it skips Xcode rebuilds, verifies code signature plus current bundle id/version/build, installs, verifies installed metadata, and runs the launch crash-watch. If a confirmed unattended build should try Xcode first and only then swap to the signed artifact, use `CONFIRM_PHYSICAL_DEVICE_UPDATE=YES make apple-device-full-entitlement-fallback-install APPLE_DEVICE_ID=<id> APPLE_DEVICE_SIGNED_ARTIFACT_PATH=<InteractiveReader.app>`. Do not use the entitlement-stripping fallback when validating iCloud features; that fallback now also requires `APPLE_DEVICE_ALLOW_ENTITLEMENT_STRIPPING=YES`.
- Run the test suite via `pytest` from the repository root.
- Language catalogs use `modules/language_constants.py` as the source of truth for backend/Web/Apple supported language names and codes. Run `make generate-language-catalogs` after editing the backend map, and `make check-language-catalogs` or `make test-apple-contracts` to verify `modules/shared/assets_data.json`, `web/src/constants/languageCodes.ts`, and `ios/InteractiveReader/InteractiveReader/Features/Shared/PlayerLanguageFlagResolver.swift` are current.
- Apple Create readiness targets use `E2E_ENV_FILE`, defaulting to `.env` when present and `.env.local` otherwise, before running XCUITest (`make test-e2e-ipad-create-readiness`, `make test-e2e-iphone-create-readiness`, `make test-e2e-tvos-create-readiness`). Shared-pipeline shortcuts are `make apple-pipeline-ipad-create-readiness[-dry-run]` and `make apple-pipeline-tvos-create-readiness[-dry-run]`.
- Shared-pipeline aggregate gates are non-physical: `make verify-apple-shared-pipeline` skips source-sync for day-to-day local checks, while `make verify-apple-golden-pipeline` runs `apple-pipeline-source-sync` first and then the same shared gate after the Mac Studio/runtime checkout is expected to match.
- The shared `basic_playback` journey includes a tvOS-only Create reachability smoke check; JSON journey steps can use `platforms` (`web`/`browser`, `iPhone`, `iPad`, `tvOS`) so one journey stays reusable across Web and Apple surfaces.
- Keep the explicit `torch>=2.0,<2.11` dependency bound unless Docker resolver evidence shows a better range; newer Linux aarch64 Torch wheels pull CUDA packages and bloat backend rebuilds.
- Library storage defaults to `/Volumes/Data/Video/Library`; override via `library_root` in `config/config.local.json` or the `LIBRARY_ROOT` environment variable when running services locally.

## Style Reminders
- Prefer explicit imports and keep module-level side effects minimal.
- Align with the FastAPI conventions already present: Pydantic models in `modules/webapi/schemas/`, services under `modules/services/`, and routers under `modules/webapi/`.
- Frontend components follow a co-located CSS Modules pattern (`Component.tsx` with `Component.module.css`).
